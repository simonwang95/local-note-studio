# Local Note Studio 待办清单

更新时间：2026-06-23

本文档是项目待办事项的唯一维护入口。`docs/progress.md` 记录已经完成的能力，`docs/development-plan.md` 只描述阶段方向，不再分别维护重复清单。

## 已完成基线

- 已有 Tauri 桌面界面、启动依赖检查、命令预览、流式日志和任务取消。
- 已支持 B站单视频、收藏夹限量测试、动态/充电动态、UP 主图文批量、微信公众号/网页、文档整理、论文速读、AI-Chat JSON、本地视频/音频和递归 EPUB 导出。
- 已支持字幕来源切换、递归媒体扫描、关键帧图文、对话检测与角色标注、A股术语校验、OCR、原始字幕保留、同名覆盖和单文件自定义文件名。
- 已支持从指定 Chrome Profile 刷新 B站 Cookie；Cookie 和个人资料路径默认隐藏。
- 转换草稿使用系统临时目录，正式目录只接收 Qwen 整理后的 Markdown 和图片资产；需要保留原文的任务会在笔记末尾附完整原文。

## P0：稳定性与可验证性

### T-001 建立自动化回归测试

现状：当前主要依靠真实任务和手工检查，容易在修改公共 worker 后影响其他任务。

目标：为 worker 请求映射、输出目录、文件命名、覆盖策略、原文保留、图片相对路径和取消流程建立可重复测试。

验收标准：

- 使用本地小型 fixture 覆盖主要任务，不依赖真实用户目录。
- 至少覆盖 B站/本地视频、网页、文档、论文、UP 主图文和 EPUB 的命令契约。
- 每次提交前可用一个命令完成 TypeScript 构建、Python 语法检查和核心回归。

### T-002 完善 B站收藏夹/系列操作

现状：已支持通过 `BILIBILI_FAV_MEDIA_ID` 和处理数量运行，但仍需手工配置收藏夹 ID。

目标：在界面中读取并选择收藏夹/系列，显示总数、已处理数、失败数和当前视频，并支持只重试失败条目。

验收标准：

- Cookie 有效时可从界面选择收藏夹，而不是编辑 `env.local`。
- 批量日志按 `当前/总数` 显示转录与 Qwen 阶段。
- 单条失败不阻断其余条目，结束后能查看并重试失败列表。

### T-003 加强 B站受限内容诊断

现状：Cookie 刷新和登录态校验已可用，充电动态走专用接口，但 B站风控、档位权限、Cookie 失效和 HTTP 412 的日志仍可能混在一起。

目标：分别识别“未登录”“账号无对应充电权限”“接口风控/412”“动态正文为空”和“视频元数据抓取失败”。

验收标准：

- 错误日志给出明确分类和下一步操作，不笼统提示重新导出 Cookie。
- 充电视频和充电动态各有至少一个授权样例的回归记录。
- Cookie 刷新后能独立验证登录态和目标内容权限。

### T-004 增加输出完整性检查

现状：各任务已经统一写入“本次输出目录”，但完整性主要靠人工打开 Markdown 检查。

目标：任务完成前检查正式笔记、来源追溯、Qwen 整理、应保留的完整原文以及图片资源是否齐全。

验收标准：

- 正式笔记不得包含系统临时草稿路径。
- Markdown 中的本地图片相对路径均可解析。
- 网页/文档/论文任务缺少完整原文或全文翻译时明确失败或警告。
- 视频是否保留原始字幕严格服从界面选项。

## P1：日常使用体验

### T-101 任务历史与恢复

- 使用 SQLite 或等价本地存储保存任务参数、状态、开始/结束时间、日志和输出文件。
- 支持查看历史、重新运行、重试失败步骤，并区分“已取消”和“失败”。

### T-102 输出快捷操作

- 任务完成后提供“打开 Markdown”“在 Finder 中显示”“复制输出路径”。
- 批量任务提供结果列表，而不是只显示最后一个文件。

### T-103 Manifest 与索引状态界面

- 展示 `video-manifest.json`、source manifest 和 B站增量状态。
- 支持按来源检查已处理、跳过、失败和需要重建的条目。

### T-104 OCR 长任务进度

- 扫描版 PDF 和图片批量任务显示页码/文件进度。
- 区分 Qwen 视觉识别、macOS Vision、Tesseract/Poppler 回退及失败原因。
- 对超长扫描 PDF 提供中断后的续跑策略。

### T-105 提升关键帧图文质量

- 根据结构化正文语义选择和放置关键帧，而不只按时间抽样。
- 去除视觉近似帧、黑屏和转场帧。
- 在 manifest 中记录帧时间、来源视频和资源路径，支持重新生成与清理。

### T-106 浏览器态网页采集

- 支持需要登录、JavaScript 渲染或反爬校验的普通网页。
- 明确区分静态 HTTP 抓取与浏览器会话抓取，避免隐式读取无关站点 Cookie。

### T-107 长批次参数调优

- 使用真实长视频、递归媒体目录和多篇 UP 主图文验证超时、重试、冷却和分块参数。
- 给出稳定的默认值，并允许按任务覆盖而不直接修改 `env.local`。

### T-108 应用托管运行环境

- 采用混合模式：`.app` 保持轻量，Python、固定版本依赖和命令行工具由应用安装到 `~/Library/Application Support/Local Note Studio/`。
- 管理可迁移的 Python 运行时和锁定依赖；管理 `ffmpeg` / `ffprobe`，允许 `yt-dlp` 独立更新，并在 EPUB 功能首次使用时按需安装 `pandoc`。
- ASR 引擎由应用环境提供；Whisper 等模型作为应用管理的可选资源下载或选择，存放在独立模型目录，不直接打进 `.app`。
- LLM 与多模态 OCR 继续使用用户配置的 OpenAI-compatible API，不内置本地大模型服务。
- 支持首次安装、依赖校验、版本升级、损坏修复、卸载和磁盘占用查看。
- 保留“使用现有 conda 环境”高级模式，并允许在托管环境与 conda 之间切换。

验收标准：

- 一台没有 conda 和 Homebrew 的新 Mac 能通过应用完成运行时初始化。
- 除 LLM/OCR API 服务、B站账号 Cookie 和按需 ASR 模型外，主要任务不要求用户手工安装命令行依赖。
- 环境版本可追踪，更新失败可以回滚或修复，不污染系统 Python。

### T-109 macOS 日常安装包

- 以 T-108 托管环境完成为前置条件，完成 release 构建、应用图标、版本信息和升级说明。
- 梳理签名、公证、首次启动权限和 Application Support 数据迁移。
- 在无开发工具、无 conda、无 Homebrew 的干净 macOS 用户环境完成端到端验收。

## P2：扩展能力

### T-201 命名配置档案

- 将当前 `localStorage` 设置升级为显式本地配置档案。
- 支持保存多组 conda、LLM、Cookie、ASR 模型和输出根目录组合。

### T-203 旧版 Office 二进制格式

- 在有可靠本地转换链路时支持 `.xls` 和 `.ppt`。
- 转换不可用时给出另存为 `.xlsx` / `.pptx` 的明确提示。

### T-204 可选 B站二维码登录

- 在不替代现有 Chrome Profile 刷新方式的前提下，评估二维码登录和 Cookie 安全存储。
- 明确账号权限、过期刷新和退出登录行为。

### T-205 本地任务队列

- 在任务历史稳定后增加排队、暂停和有限并发。
- 视频缓存和输出写入必须按任务隔离，禁止批量任务串台。

## 暂不纳入

- 云同步、在线账号系统和多人协作。
- 在 `.app` 安装包中直接捆绑大型 LLM/ASR 模型；ASR 模型允许由应用按需管理。
- 在本地工作流稳定前进行应用商店发布。
